＝＝ このフォルダについて ＝＝

1.はじめに
このフォルダには、RTミドルウェア講習会 第２部 「RTミドルウェアの使い方」で紹介したRTコンポーネントと、それに必要なRTミドルウェアのツール（ネーミングサービス、RTシステムエディタ）を収録したものです。

ここに収録されているコンポーネント群は、RTミドルウェアをPCにインストールせずに動作させることができるようにしています。

RTミドルウェアを使った簡単なシステム例で、その動作を確認していただければ幸いです。


2. このフォルダの構成
このフォルダは、下記のような構成になっています。


./-+
|
+- OpenRTM-aist -- 1.1 -+- omniORB (CORBAライブラリ）
|                       +- tools （RTシステムエディタ、ネームサービス）
|                       +- runtime（RTミドルウェアの実行ライブラリ）
|
+- OpenHRI-Demo (音声対話コンポーネントを使ったデモシステム）
|
+- src (上記のデモシステムで使用したRTコンポーネントのソースファイル）

3．収録されているデモシステム

このフォルタ"には、コミュニケーション知能モジュール群 OpenHRIを用いた簡単なデモシステムを収録しています。

openHRI-Demoの下の0 SimpleDemoのフォルダに.起動、初期設定用のバッチファイルがありますので、コンポーネントの起動、コンポーネント間の接続、アクティベーションを簡単に実行できるようになっております。

4.SimpleDemoの実行
このUSBメモリに収録しておりますSimpleDemoは、OpenHRIの対話エンジンSEATを使ってPowerPointとビデオビューアをGUIを使って動かす筒単なものです。GUIの代りに音声認識モジュールを使ったデモシステムは、このシステムのGUIコンポーネントを置き換えることで実現することができます。

以下に、SimpleDemoの実行方法を示します。

まず、RTコンポーネントを使ったシステムで必要なネーミングサービスを起動します。(\Demo-hara\rtm-naming.bat)
必要に応してRTシステムエディタも起動しておきます。(\Demo-hara\RTSE.bat)
この2つのプログラムの実行のためのバッチファイルも収録しておりますので、それらをダブルクリックで起動します。

次に、0 SimpleDemoのフォルダ(\Demo-hara\OpenHRI-Demo\0 SimpleDemo)に移動して、コンポーネント起動(0 StartDemo.bat)、コンポーネント間の接続(1 ConnectRTC.bat)、コンポーネントのアクティベーション(2 ActivateRTC.bat)の順で、バッチファイルを起動して下さい。

RTシステムエディタで、すべてコンポーネントが起動していること、接続ができていて、アクティベーションされていることを確認してください。

以上でシステムの準備は、完了ですので、GUIパネルを使ってPowerPointとビデオを動作させてみて下さい。

システムの終了は、コンポーネントのディアクティベーション（3 DeactivateRTC.bat）、コンポーネント間の接続解除（4 DisconnectRTC.bat）、コンポーネントの終了（5 DemoExit.bat）の順で終了手続を行って下さい。
